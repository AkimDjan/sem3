### Продолжаем

Для синхронизации значений переменных, временно хранящихся в регистрах и кэш-памяти текущей нити, с основной памятью  применяется директива flush.
```c
#pragma omp flush [(список)]
```
Рекомендуется в тех местах, где нужно синхронизировать там, где необходимо

Для барьерной синхронизации нитей применяется директива barrier.
```c
#pragma omp barrier
```
При директиве barrier неявно вызывается директива flush для всех переменных.

Неявно вызывается в конце *parallel*, *for*, *sections* и *single* без опции nowait

Любая барьерная синхронизация - дополнительные барьерные расходы

Для определения критической области применяется директива critical.
```c
#pragma omp critical [(имя)]
```
Применяется внутри параллельной секции.

Действует на оператор или блок операторов. Побочные входы и выходы из критической секции запрещены. В конце и в начале каждой критической секции неявно выполняется директива flush для всех переменных

Для определения атомарности выполнения операции применяется директива atomic.
```c
#pragma omp atomic [опция]
```
Без опций действует на один оператор. С опциями – на один оператор или на блок из двух операторов.
